# 游戏攻略聚合平台 - 内容添加规则

本规则必须严格遵循，添加游戏和文章时逐项检查。

---

## 一、添加游戏（Games）

### 1.1 必填字段

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| name | VARCHAR(100) | 唯一，不可重复 | 游戏中文全名，如"原神""和平精英" |
| slug | VARCHAR(100) | 唯一，不可重复 | 英文短标识，小写+连字符 |
| description | TEXT | 不可为空 | 游戏详细介绍，2-5句话 |
| category | VARCHAR(50) | 必须来自已有分类列表 | 游戏类型 |
| developer | VARCHAR(100) | 不可为空 | 开发商名称 |
| publisher | VARCHAR(100) | 不可为空 | 发行商名称 |
| cover_image | VARCHAR(500) | 遵循路径规范 | 游戏封面图路径 |
| release_date | DATETIME | datetime对象，非字符串 | 发行日期 YYYY-MM-DD |

### 1.2 Slug 命名规范

- 全部小写英文字母
- 单词间用连字符 `-` 分隔
- 禁止数字开头、禁止特殊符号
- 示例：
  - 原神 → `genshin-impact`
  - 和平精英 → `pubg-mobile`
  - 崩坏：星穹铁道 → `honkai-star-rail`
  - 重返未来1999 → `reverse-1999`

### 1.3 游戏分类（category）

已有分类（只能从以下选择，不要自创）：

```
RPG, MOBA, ARPG, 塔防, MMORPG, 大逃杀, FPS, 策略, 战棋,
开放世界, 冒险, 生存冒险, 非对称竞技, 派对游戏, 卡牌RPG, 女性向
```

### 1.4 封面图规范

- 路径格式：`/static/games/game_{id}.jpg`
- 文件存放位置：CVM `/data/game-guide-platform/static/games/`
- 图片来源优先 Steam `library_600x900_2x.jpg`（600x900竖版），其次 `header.jpg`（460x215）
- 非Steam游戏去官网 `og:image` 获取
- 严禁外链URL，必须本地化存储

### 1.5 Description 规范

- 2-5句话，介绍游戏核心玩法、特色、开发商
- 中文描述，语气正式但不死板
- 不要照搬官方简介，要有概括性

### 1.6 添加流程（操作步骤）

```
1. 确定游戏中文名和slug（slug必须唯一，先查数据库确认不重复）
2. 下载封面图到本地 /static/games/ 目录（等插入后才知道ID，分两步）
3. 通过后端API或sqlite3插入游戏记录
4. 查询刚插入记录的id，将封面图命名为 game_{id}.jpg
5. 更新数据库 cover_image 字段为 /static/games/game_{id}.jpg
```

## 二、添加文章（Articles）

### 2.1 必填字段

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| title | VARCHAR(200) | 不可为空 | 文章标题 |
| slug | VARCHAR(200) | 唯一，不可重复 | 英文短标识 |
| content | TEXT | 不可为空，必须Markdown格式 | 文章正文 |
| author_id | INTEGER | 必须为有效用户ID | 作者ID，默认设为 2 |
| category | VARCHAR(50) | 只能 guide/news/tips | 文章分类 |
| game_id | INTEGER | 必须为有效游戏ID | 关联的游戏 |
| tags | VARCHAR(200) | 逗号分隔，可为空 | 标签 |
| cover_image | VARCHAR(500) | 遵循路径规范 | 通常用关联游戏的封面 |
| is_published | BOOLEAN | 默认 1 (true) | 是否发布 |

### 2.2 Slug 命名规范

- 全部小写英文+连字符
- 格式：`{游戏slug}-{内容主题}-{序号}`
- 示例：
  - 原神新手攻略 → `genshin-impact-beginner-guide-1`
  - 和平精英上分技巧 → `pubg-mobile-ranking-tips-2`
  - 碧蓝航线科研攻略 → `azur-lane-systems-guide-23`

### 2.3 文章分类

只有三种，严格区分：

- **guide**（攻略）：游戏玩法、技巧、教程、角色培养、装备推荐等指导性内容
- **tips**（技巧）：短小精悍的技巧点、小贴士、冷知识
- **news**（资讯）：游戏更新公告、活动通知、版本情报

### 2.4 文章内容格式规范（核心规则）

所有文章正文 content 必须使用 Markdown 格式，要求结构清晰、层次分明。

#### 2.4.1 标题层级

```markdown
## 一、大段落标题（使用 H2 ##）
### 1. 子段落标题（使用 H3 ###）
```

- 文章内不要用 H1（#），H1 留给页面标题
- H2 用于主分段，一个主分段内最多5个子分段
- 标题编号统一用中文数字：一、二、三... 或 1. 2. 3.

#### 2.4.2 表格

对比类、清单类内容必须使用表格：

```markdown
| 列1 | 列2 | 列3 |
|------|------|------|
| 数据 | 数据 | 数据 |
```

适用场景：角色对比、装备推荐、资源优先级、属性一览

#### 2.4.3 列表

```markdown
- 无序列表项（用于并列关系）
- 每项一句话

1. 有序列表项（用于步骤、排序）
2. 第一步
3. 第二步
```

#### 2.4.4 强调与提示

```markdown
**重点内容** 用粗体标注关键信息

> 提示、注意、警告类的内容用引用块
> 可以多行
```

#### 2.4.5 代码/术语

```markdown
`游戏术语` 用反引号标注
```

#### 2.4.6 内容结构要求

每篇文章必须包含：

1. **开篇概述**：2-3句话说明文章解决什么问题、适合什么读者
2. **核心内容**：3-5个主分段（H2），每段有实质性内容，不能一句话带过
3. **总结/建议**：末尾给出总结或最终建议

#### 2.4.7 格式检查清单

- [ ] 是否使用了 H2（##）作主分段？
- [ ] 对比类内容是否用了表格？
- [ ] 步骤类内容是否用了有序列表？
- [ ] 关键信息是否用粗体标注？
- [ ] 是否有开篇概述和结尾总结？
- [ ] 分段是否清晰，避免一大段文字堆砌？

#### 2.4.8 反面示例（不要这样写）

```markdown
少前2的高难关卡（深渊、遭遇战）是检验队伍强度的试金石。

深渊模式每层有特定敌人组合和场地DEBUFF。打深渊前先看敌方配置...
```
问题：无标题分段、无表格、纯文字堆砌、没有结构层次。

#### 2.4.9 正面示例

```markdown
## 一、前言

少前2的高难关卡（深渊、遭遇战）是检验队伍强度的试金石。本文介绍深渊模式的核心打法和BOSS战通用策略。

## 二、深渊模式机制

| 层级 | 敌人类型 | 推荐队伍 |
|------|----------|----------|
| 1-3层 | 突击兵为主 | 桑朵莱希 + 寇尔芙 |
| 4-6层 | 精英兵+BOSS | 桑朵莱希 + 奇塔 + 医疗 |

## 三、BOSS战通用打法

### 1. P1阶段：清小怪
- BOSS旁边的小怪会给BOSS上BUFF，不清掉BOSS防御极高
- 优先击杀顺序：医疗兵 > 狙击手 > 突击兵

### 2. P2阶段：集火BOSS
**关键提示：BOSS的AOE技能覆盖全场，必须全员找掩体躲避。**
- 看到红色预警圈立即移动
- 桑朵莱希的技能留着打断BOSS读条

## 四、总结

深渊的关键在于理解每层的敌人组合，针对性配队。P1清小怪、P2躲技能是通关的基本操作。
```

#### 2.4.10 内容长度要求（关键规则）

攻略类文章（category=guide）内容必须 **3000字符以上**。这不是硬性数字指标，而是确保内容有足够深度：

- 新手攻略：覆盖角色选择、核心机制、养成路线、资源规划、常见误区，至少4个主分段
- 进阶攻略：覆盖高阶机制、数值分析、阵容搭配、不同场景策略，至少4个主分段
- BOSS/关卡攻略：覆盖机制拆解、分阶段打法、推荐配置、翻车原因分析，至少4个主分段
- 系统详解：覆盖基础机制、进阶应用、数据表格、优化策略、常见问题，至少5个主分段

技巧类（tips）和资讯类（news）不做字数要求，但技巧也必须结构清晰。

累计字数不足3000的攻略需要补充额外章节。参考已被重写的25篇攻略（ID 75-99，每篇2766-3144字）作为格式和深度标准。

### 2.5 Tags 标签规范

- 逗号分隔，不超过5个
- 全部中文
- 示例：`新手,攻略,入门,原神` 或 `配队,技巧,进阶`
- 不要放slug、不要放英文

### 2.6 封面图

- 通常使用关联游戏的封面图：`/static/games/game_{game_id}.jpg`
- 如果文章有专属封面图，存到 `/static/articles/article_{id}.jpg`

### 2.7 author_id

- 直接通过 sqlite3 插入时，author_id 必须设为有效值
- 默认使用 `author_id = 2`（系统编辑账号）
- 禁止 author_id 为 NULL（会导致API 500错误）

## 三、添加角色（Characters）

### 3.1 必填字段

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| game_id | INTEGER | 必须为有效游戏ID | 所属游戏 |
| name | VARCHAR(100) | 不可为空 | 角色中文名 |
| description | TEXT | 不可为空 | 角色介绍 |
| rarity | VARCHAR(20) | 可选 | 稀有度，如 ★★★★★ |
| element | VARCHAR(30) | 可选 | 属性/元素 |
| weapon_type | VARCHAR(30) | 可选 | 武器类型 |
| image | VARCHAR(500) | 可选 | 角色立绘图路径 |

### 3.2 图片路径

- 格式：`/static/characters/char_{id}.jpg`
- 存放位置：CVM `/data/game-guide-platform/static/characters/`
